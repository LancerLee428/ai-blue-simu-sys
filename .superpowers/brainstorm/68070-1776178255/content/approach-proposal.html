实现路径选择

三种复杂度不同的实现路径，权衡开发成本与功能深度

A

快速 MVP

AI 输出 JSON → 前端解析渲染 → 阶段手动控制

  • AI prompt 模板 + JSON Schema 约束输出
  • 路由/探测范围/打击点全部以 Cesium 静态 graphics 渲染
  • 阶段切换触发实体位置插值动画
  • 预计开发时间：短
B

结构化方案引擎

引入 Scenario DSL + 专用的 Cesium Graphics Layer

  • 定义战术方案数据模型（包含路线、范围、打击事件）
  • Cesium Entity + Primitive 混合渲染层
  • 轻量执行引擎：定时器驱动阶段推进
  • 预计开发时间：中
C

完整仿真平台

态势数据模型 + 仿真引擎 + 事件系统

  • 完整的态势-事件-状态机模型
  • 实体移动物理引擎（速度、加速度、燃油消耗）
  • 探测/打击判定逻辑（距离检测、命中率）
  • 预计开发时间：长

推荐 B

  • A 太简陋，后续扩展代价高
  • C 投入产出比低，先验证价值再投入
  • B 有边界清晰的架构，可渐进演进

B 的边界

  • 执行引擎不含物理仿真（移动是插值，非物理）
  • 打击判定为规则驱动，非概率模型
  • 可演进到 C，但不需要一开始就做
