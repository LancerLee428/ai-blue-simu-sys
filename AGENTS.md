# Codex 项目规范

## 代码交付规范

### 构建验证
- **所有代码变更必须先执行 `npm run build` 验证无误后再交付**
- 构建失败时必须修复所有错误后才能交付
- 不允许交付包含构建错误的代码

### 代码质量
- 修复语法错误、类型错误、重复声明等问题
- 确保代码可以立即运行，无需用户手动修复

## 性能优化规范

### 动画和定时任务
- **禁止使用 `setTimeout`、`setInterval` 等定时器**
- 所有动画变更统一使用 `requestAnimationFrame` (RAF)
- 需要节流的操作在 RAF 循环中实现节流逻辑

### 状态更新
- 避免高频率的状态更新（如每帧更新 Store）
- 使用节流机制降低更新频率（如 100ms 更新一次）
- 批量更新状态，避免多次触发响应式更新

## 数据流规范

### 单向数据流
- 保持清晰的单向数据流：数据源 → 桥接层 → 状态管理 → UI
- 避免双向绑定导致的状态混乱
- 状态变化必须有明确的触发源

### 状态同步
- 引擎状态变化通过回调机制同步到 Store
- 立即同步：状态变化（play/pause/stop）、用户操作（倍速切换）
- 节流同步：高频更新（时间、进度）

## 项目架构

### 核心服务
- `ExecutionEngine`: 推演引擎，负责动画循环和状态管理
- `MapRenderer`: 地图渲染器，负责 Cesium 可视化
- `ActionPlanStore`: 行动方案状态管理

### 数据流示例
```
用户操作 → ExecutionEngine → WorkbenchView (桥接) → ActionPlanStore → UI 组件
```
