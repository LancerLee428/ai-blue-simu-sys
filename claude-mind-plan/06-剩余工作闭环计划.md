# 剩余工作闭环计划：武器、爆炸与态势效果补齐

> 基于 `01-现状分析.md` 到 `05-实施路线图.md`，对当前系统已完成内容、仍缺内容、实施顺序和验证口径做一次收敛。后续实施必须先验证，再上报。

## 1. 当前已完成范围

### 1.1 已完成：武器系统最小闭环

- 已扩展武器相关类型：`WeaponType`、`GuidanceType`、`WeaponSpec`、`Weapon`、`WeaponLaunchEvent`、`WeaponImpactEvent`。
- 已新增武器数据库：`apps/web/src/services/weapon-database.ts`。
- 已新增武器系统：`apps/web/src/services/weapon-system.ts`，可根据 `attack` 事件推导发射、飞行、命中。
- 已接入推演引擎：`ExecutionEngine` 在 RAF 循环中计算武器飞行状态，并触发 `weapon-launch` / `weapon-impact`。
- 已接入地图渲染：`MapRenderer.syncWeapons()` 渲染导弹实体和轨迹。
- 已补齐生成链路：LLM JSON / XML 导入后会经 `normalizeTacticalScenario()` 补齐 `loadout`、`strikeTasks`、`attack` 事件和 `visualEffects`。
- 已补齐 XML 表达：`Loadout`、`StrikeTasks`、`VisualEffects` 可解析和导出。

### 1.2 已完成：爆炸效果最小闭环

- 已定义 `VisualEffectsConfig`，XML 可配置武器尾迹和爆炸视觉参数。
- 已支持三类命中效果：`missile-impact`、`ship-impact`、`ground-impact`。
- 已在武器命中时触发爆炸，爆炸位置来自打击目标对应的 `StrikeTask`。
- 示例 XML 已调整为可演示时间轴：导入后点击推演，可看到武器轨迹和三类命中爆炸。

### 1.3 已部分完成：尾焰 / 轨迹

- 已用 `PolylineGlowMaterialProperty` 渲染飞行轨迹。
- 已用 Canvas 绘制武器图标和尾焰颜色。
- 但 XML 配置目前是单色模板，还没有完整实现“助推 / 巡航 / 末端”阶段颜色自动切换和尾焰长度随速度变化。

## 2. 仍缺内容

### P0 缺口 1：武器系统还不是设计文档里的完整版本

缺少内容：

- 缺少 `launch()` / `update()` 风格的显式生命周期接口。
- 缺少比例导引或其他制导算法，目前是从发射点到目标点的简化插值。
- 缺少命中判定窗口、脱靶、拦截、失效等结果分支。
- 武器数据库还需要补齐 XML 已引用但数据库未覆盖的型号，例如 `hhq-9`、`rocket`。
- XML/LLM 生成链路还没有强校验“攻击事件使用的 weaponId 必须存在于武器数据库”。

### P0 缺口 2：爆炸效果还不是设计文档里的完整版本

缺少内容：

- 缺少独立 `ExplosionRenderer` 服务。
- 缺少 `Explosion`、`ExplosionEvent` 运行时实体类型。
- 缺少烟雾、碎片、闪光效果。
- 爆炸半径目前来自 XML 配置，不是由战斗部重量 / 伤害值计算得出。
- 爆炸持续时间目前约 1 到 1.4 秒，尚未达到设计里的 3 到 5 秒专业表现。
- 缺少同时爆炸数量限制和生命周期管理。

### P0 缺口 3：损伤评估系统完全缺失

缺少内容：

- `EntitySpec` 没有 `health`、`maxHealth`、`armor` 等战损字段。
- 缺少 `DamageEvent`、`EntityDestroyedEvent`。
- 缺少 `DamageCalculator`。
- 命中后只展示爆炸，不会计算目标受损 / 摧毁。
- UI 没有展示生命值、战损统计、战斗日志。

### P1 缺口 4：雷达扫描动画缺失

缺少内容：

- 当前探测范围是静态圆 / 半球 / 锥形可视化。
- 缺少 `SensorAnimationRenderer`。
- 缺少旋转扫描波束、扫描速度配置、多个雷达并行动画。
- XML 暂无 `SensorEffects` / `RadarScanEffects` 配置。

### P1 缺口 5：电子战可视化缺失

缺少内容：

- 当前有电子战逻辑和干扰范围计算，但缺少地图上的干扰波纹。
- 缺少 `EWVisualizationRenderer`。
- XML 暂无 `ElectronicWarfareEffects` 配置。
- 推演过程中还没有持续渲染干扰机脉冲。

### P1 缺口 6：尾焰效果仍需增强

缺少内容：

- 缺少独立 `WeaponTrailRenderer`。
- XML 配置没有表达不同飞行阶段的尾焰颜色。
- 尾焰长度 / 粗细没有和速度、阶段强关联。

### P2 缺口 7：性能与用户开关缺失

缺少内容：

- 缺少视觉效果总开关和细分开关。
- 缺少性能模式：高 / 中 / 低。
- 缺少动画 LOD、最大活跃爆炸数、最大扫描动画数。
- 缺少可选相机跟随导弹、慢动作回放。

## 4. 总验证清单

每个阶段完成后必须执行：

```bash
npm run build
```

涉及生成链路 / XML 的阶段还必须执行：

```bash
npx tsx --test apps/web/src/services/__tests__/scenario-generation-chain.test.ts apps/web/src/services/__tests__/weapon-system.test.ts
xmllint --noout data-example/东海联合打击-2024-1777165955760.xml
```

新增专项服务后必须增加并执行对应单测：

```bash
npx tsx --test apps/web/src/services/__tests__/<对应测试文件>.test.ts
```

最终上报前必须确认：

- 构建通过。
- 单测通过。
- 示例 XML 语法通过。
- 示例 XML 导入链路包含 `Loadout`、`StrikeTasks`、`VisualEffects`。
- 推演可演示至少：武器飞行、命中爆炸、损伤状态变化、雷达扫描、电子战波纹。

## 5. 推荐执行顺序

1. 阶段 A：先修武器数据库和武器链路可靠性。
2. 阶段 B：再抽出爆炸渲染器，补专业爆炸效果。
3. 阶段 C：接入损伤评估，让命中影响战斗状态。
4. 阶段 D：补雷达扫描动画。
5. 阶段 E：补电子战波纹。
6. 阶段 F：补尾焰增强和性能开关。

这份顺序符合 `05-实施路线图.md` 的原则：先纵向闭环，再横向增强；先功能正确，再视觉打磨。
