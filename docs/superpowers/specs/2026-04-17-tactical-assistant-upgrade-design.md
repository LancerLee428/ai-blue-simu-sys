## 1. 实体形状渲染系统

### 目标

| 状态 | 尺寸倍数 |

- 修改 `cesium-graphics.ts`，新增形状映射配置
- 修改 `map-renderer.ts`，根据实体类型设置对应几何形状
- 保持现有颜色方案（红蓝双方）

## 2. Word 导入导出系统

### 2.1 导出格式

#### 文档结构

```
# 作战方案

## 方案概述
[summary]

## 战术分析
[AI 生成的思维链内容]

## 1. 兵力编成

### 红方（中国）
- 兵力名称
  - 实体名称 | 类型 | 初始位置 | 状态

### 蓝方（日本）
- 兵力名称
  - 实体名称 | 类型 | 初始位置 | 状态

## 2. 阶段划分

### 阶段 1: [name]
**时间**: T+00:00 ~ T+[duration]秒
**描述**: [description]

**关键事件**:
- T+[timestamp]秒: [detail]

**兵力状态**:
- [entityName]: [status] @ [position]

### 阶段 2: ...

## 3. 进攻路线
[实体名称] -> [路径点列表]

## 4. 探测范围
[实体名称]: 半径 [radiusMeters]米

## 5. 打击任务
- T+[timestamp]秒: [attackerName] 攻击 [targetName]

## 元数据
- 生成时间: [generatedAt]
- 模型: [modelUsed]
- 置信度: [confidence]
```

#### 技术实现

- 使用 `docx` 库生成 Word 文档
- 表格形式展示兵力编成
- 时间轴用列表展示
- 导出按钮在 AIAssistantPanel 顶部

### 2.2 导入解析

#### 约束条件

- **格式要求**: 必须是系统导出的标准格式
- **必需章节**: 兵力编成、阶段划分、打击任务
- **解析容错**: 缺失可选章节时提供警告，继续解析

#### 解析流程

1. 读取 Word 文档内容
2. 按标准章节结构解析
3. 提取实体数据 → `EntitySpec[]`
4. 提取阶段数据 → `Phase[]`
5. 提取路线/探测/打击数据
6. 构造 `TacticalScenario` 对象
7. 验证数据完整性
8. 渲染到地图

#### 技术实现

- 使用 `mammoth.js` 提取 Word 文本
- 正则表达式匹配章节结构
- 解析表格/列表提取数据
- 错误处理：格式不符时提示用户

## 3. 战术约束引擎

### 3.1 地域约束

#### 红方（中国）
```javascript
// 允许区域
- 中国大陆: long 73-135, lat 18-53
- 台湾岛: long 119-122, lat 21-25
- 相关领空: altitude 0-15000m
```

#### 蓝方（日本）
```javascript
// 允许区域
- 日本本土: long 129-145, lat 30-46
- 冲绳群岛: long 127-129, lat 25-27
- 相关领空: altitude 0-15000m
// 允许活动海域
- 东海、台海等公海区域
```

#### 禁止规则
- 红方实体不得出现在日本陆地
- 蓝方实体不得出现在中国大陆或台湾岛
- 允许双方在公海、国际领空活动

### 3.2 攻防逻辑约束

#### 进攻路线规则
- 每条路线必须有明确的 `targetEntityId`
- 路线的终点必须接近目标位置（±50km 内）
- 红方为主攻方，路线方向应指向蓝方
- 蓝方为防守方，路线应在防御区域内

#### 探测范围规则
- 探测中心必须关联到具体实体
- 探测半径必须覆盖潜在威胁方向
- 红方探测范围应覆盖蓝方可能的进攻路径
- 蓝方探测范围应覆盖本土防御方向

### 3.3 兵力分散规则

#### 最小间距约束
```javascript
// 同一方任意两个实体间距 ≥ 10km
function checkEntitySpacing(entities: EntitySpec[]): boolean {
  for (let i = 0; i < entities.length; i++) {
    for (let j = i + 1; j < entities.length; j++) {
      const dist = calculateDistance(entities[i].position, entities[j].position);
      if (dist < 10000) return false; // 10km
    }
  }
  return true;
}
```

#### 编队聚集规则
- 同一编队（如航母战斗群）实体间距可缩小至 2-5km
- AI 生成时标记 `formationId` 字段
- 同一编队的实体可以靠近部署

### 3.4 AI Prompt 增强

在 `SYSTEM_PROMPT` 中新增约束条款：

```
**地域约束**：
- 红方（中国）只能部署在中国大陆、台湾岛及其领空
- 蓝方（日本）只能部署在日本本土、冲绳及其领空
- 禁止任何一方出现在对方陆地上
- 双方可在公海、国际领空活动

**攻防逻辑约束**：
- 进攻路线必须有明确的攻击目标（targetEntityId）
- 路线终点必须在目标实体 50km 范围内
- 探测范围必须覆盖对方可能的威胁方向

**兵力分散规则**：
- 同一方任意两个实体间距至少 10km
- 同一编队实体间距可缩小至 2-5km
- 兵力部署要清晰展示数量，不能重叠
```

### 3.5 验证流程

生成方案后执行验证：

1. **地域验证** → 检查所有实体位置是否符合地域约束
2. **逻辑验证** → 检查路线、探测是否符合攻防逻辑
3. **分散验证** → 检查实体间距是否符合最小间距
4. **完整性验证** → 检查必需字段是否齐全
5. 验证失败时提示 AI 重新生成

## 5. 推演倍速系统

### 5.1 倍速实现

#### 时间缩放逻辑

```javascript
// ExecutionEngine 中
class ExecutionEngine {
  private speed: number = 1;
  private lastRealTime: number = 0;

  setSpeed(speed: number) {
    this.speed = speed;
  }

  update(deltaTime: number) {
    const simDelta = deltaTime * this.speed;
    this.currentTime += simDelta;
    // ... 更新实体位置
  }
}
```

### 5.2 控制组件

**倍速选择器**：
```
[0.5x] [1x] [2x] [5x] [10x] [20x]
  ↑ 当前选中
```

**播放控制**：
```
[⏮ 上阶段] [⏪ 后退1秒] [▶️ 播放] [⏸️ 暂停] [⏩ 前进1秒] [⏭ 下阶段]
```
