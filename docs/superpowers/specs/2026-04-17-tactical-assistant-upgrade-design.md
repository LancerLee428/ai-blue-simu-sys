## 2. Word 导入导出系统

### 2.1 导出格式

#### 文档结构

```
# 作战方案

## 方案概述
[summary]

## 战术分析
[AI 生成的思维链内容]

## 1. 兵力编成

### 红方（中国）
- 兵力名称
  - 实体名称 | 类型 | 初始位置 | 状态

### 蓝方（日本）
- 兵力名称
  - 实体名称 | 类型 | 初始位置 | 状态

## 2. 阶段划分

### 阶段 1: [name]
**时间**: T+00:00 ~ T+[duration]秒
**描述**: [description]

**关键事件**:
- T+[timestamp]秒: [detail]

**兵力状态**:
- [entityName]: [status] @ [position]

### 阶段 2: ...

## 3. 进攻路线
[实体名称] -> [路径点列表]

## 4. 探测范围
[实体名称]: 半径 [radiusMeters]米

## 5. 打击任务
- T+[timestamp]秒: [attackerName] 攻击 [targetName]

## 元数据
- 生成时间: [generatedAt]
- 模型: [modelUsed]
- 置信度: [confidence]
```

#### 技术实现

- 使用 `docx` 库生成 Word 文档
- 表格形式展示兵力编成
- 时间轴用列表展示
- 导出按钮在 AIAssistantPanel 顶部

### 2.2 导入解析

#### 约束条件

- **格式要求**: 必须是系统导出的标准格式
- **必需章节**: 兵力编成、阶段划分、打击任务
- **解析容错**: 缺失可选章节时提供警告，继续解析

#### 解析流程

1. 读取 Word 文档内容
2. 按标准章节结构解析
3. 提取实体数据 → `EntitySpec[]`
4. 提取阶段数据 → `Phase[]`
5. 提取路线/探测/打击数据
6. 构造 `TacticalScenario` 对象
7. 验证数据完整性
8. 渲染到地图

#### 技术实现

- 使用 `mammoth.js` 提取 Word 文本
- 正则表达式匹配章节结构
- 解析表格/列表提取数据
- 错误处理：格式不符时提示用户

## 3. 战术约束引擎

### 设计原则

**让 AI 做战术决策，让代码做 GIS 计算。**

LLM 不擅长精确的经纬度计算和距离估算，因此采用三层约束体系：

```
AI 生成战术意图（部署位置/攻防关系/进近方式）
  ↓
前端后处理（自动生成路线航路点/探测范围）
  ↓
验证（失败率大幅降低）
  ↓
验证未通过 → 自动喂回错误给 AI 重试（最多 2 次）
  ↓
渲染到地图
```

### 3.1 第一层：Prompt 锚点坐标约束

不给 AI 自由的地域范围，而是给**预定义的真实军事基地/海域坐标**让 AI 选择（±0.3° 微调）。

| 嘉手纳基地（冲绳） | 127.77 | 26.36 | 战斗机、侦察机 |

### 3.2 第二层：声明式意图 + 自动后处理

#### 路线：AI 声明 → 代码插值

AI 只生成 `routeIntents`（谁打谁、进近方式）：
```json
{
  "entityId": "entity-001",
  "targetEntityId": "entity-005",
  "approach": "flanking-north",
  "side": "red"
}
```

`route-generator.ts` 自动在起止点之间插值生成 3-5 个专业航路点，保证终点就是目标位置。

支持 4 种进近模式：`direct | flanking-north | flanking-south | low-altitude`

#### 探测：AI 声明 → 代码分配半径

AI 只生成 `detectionIntents`（谁有探测能力、哪种类型）：
```json
{
  "entityId": "entity-005",
  "type": "radar",
  "side": "blue"
}
```

`detection-resolver.ts` 根据平台类型查表分配半径，并自动扩大到覆盖最近敌方实体。

### 3.3 地域验证修正

修复 bounding box 重叠问题：

```
旧: 中国 lon 73~135 ← 与日本 lon 129~145 重叠在 129~135
新: 中国 lon 73~125 ← 不再重叠
```

采用分层优先级判定（台湾 → 冲绳 → 日本 → 中国 → 公海）。

### 3.4 兵力分散规则

- 同一方任意两个实体间距 ≥ 5km（放宽到 5km，更适配海上编队）
- 探测范围验证降级为**警告**，不阻塞方案生成

### 3.5 自动重试机制

验证失败后，Store 自动将错误信息附加到原始需求中重新请求 AI，最多重试 2 次。若仍有问题则以警告形式展示，不阻塞方案。

### 3.6 涉及文件

| 文件 | 职责 |
|------|------|
| `ai-tactical.ts` | Prompt + 后处理管线 |
| `route-generator.ts` | 路线意图 → 航路点插值 |
| `detection-resolver.ts` | 探测意图 → 半径分配 |
| `tactical-validator.ts` | 地域/逻辑/间距验证 |
| `tactical-scenario.ts` (store) | 自动重试机制 |

### 4.2 数据结构

```typescript
interface ActionPlan {
  id: string;

    currentTime: number; // 当前推演时间（秒）
    speed: number; // 当前倍速
  };
}
```

## 5. 推演倍速系统

### 5.1 倍速实现

#### 时间缩放逻辑

```javascript
// ExecutionEngine 中
class ExecutionEngine {
  private speed: number = 1;
  private lastRealTime: number = 0;

  setSpeed(speed: number) {
    this.speed = speed;
  }

  update(deltaTime: number) {
    const simDelta = deltaTime * this.speed;
    this.currentTime += simDelta;
    // ... 更新实体位置
  }
}
```

### 5.2 控制组件

**倍速选择器**：
```
[0.5x] [1x] [2x] [5x] [10x] [20x]
  ↑ 当前选中
```

**播放控制**：
```
[⏮ 上阶段] [⏪ 后退1秒] [▶️ 播放] [⏸️ 暂停] [⏩ 前进1秒] [⏭ 下阶段]
```
