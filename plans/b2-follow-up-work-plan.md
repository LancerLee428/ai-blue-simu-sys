# B2 Follow-up Work Plan

## Goal
在当前“生成草案 → 确认写回”闭环基础上，补齐更接近真实业务的人机协同能力，包括重生成、拒绝、撤销确认与版本推进。

## Scope

### 1. 草案重生成
- 在 `AI Assistant` 中增加“重生成草案”动作
- 保留上一次草案摘要与对象列表，支持对比
- 后端为草案生成增加请求上下文标识，避免前端误覆盖旧结果

### 2. 草案拒绝
- 增加“拒绝草案”动作
- 拒绝后将当前草案状态标记为 `rejected`
- 前端展示拒绝原因输入框与拒绝后的状态提示

### 3. 撤销确认
- 在 `Scenario Workspace` 中支持撤销最近一次确认写回
- 撤销后恢复到确认前的想定版本与对象状态
- `Situation Workbench` 同步回滚部署点位

### 4. 版本推进历史
- 为想定工作空间增加版本历史记录
- 至少记录：版本号、动作类型、时间、对象数量变化
- 前端增加一个最小版本历史列表

## Suggested Implementation Order
1. 先做草案重生成，确保 AI 工作流具备重复操作能力
2. 再做草案拒绝，形成“通过 / 不通过”双分支
3. 然后补撤销确认，打通回滚链路
4. 最后补版本推进历史，把状态流留痕

## Verification
- `npm run build`
- 本地联调验证 AI Assistant 中 4 类动作的状态流转
- 确认 `Scenario Workspace` 与 `Situation Workbench` 联动正确

## Notes
- 当前阶段优先保持最小实现，不引入数据库
- 版本历史先放在 server 内存态，后续再迁移到持久化存储
